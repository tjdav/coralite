Getting Started

This guide will walk you through creating your first web page using Coralite, a static site generator built around HTML modules.

Coralite components represent a simpler form of web components. They allow you to create custom elements using "template" and "slot" elements, but they do not use the Shadow DOM, as they're server-side rendered and there is no hydration.

Create a new project directory with the following structure:

  my-coralite-site/
  ├── src/
  │   ├── components/
  │   │   ├── header.html
  │   │   ├── body.html
  │   │   └── footer.html
  │   └── pages/
  │       └── index.html
  └── dist/

Every component <template> must have an "id" attribute that matches its usage in pages, in this case our header component has the id of "coralite-header". Take note in the example below the use of double curly braces {{ title }} will display the attribute value

Just like Web Components can pass data to components through both attribute values and slots so can Coralite components, however, Coralite components use a special syntax with double curly braces to display the attribute values.

In order to use the component in pages, the custom element name must match the component id.

My coralite site

To manually build the example, run the following command:

coralite -c src/components -p src/pages -o dist

If you've followed the build script step on the installation guide, run the build script:

npm run build

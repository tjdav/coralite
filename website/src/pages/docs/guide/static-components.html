Static Components

Static components are the building blocks of Coralite. They are purely HTML-based elements without complex scripting logic. If a piece of UI doesn't need to fetch data, handle user events, or compute complex state, it should be a static component.

Every component in Coralite is defined inside an HTML file using the standard <template> tag. The most important part of a component is its id attribute. This ID becomes the custom HTML tag name you will use to include the component in your pages. Note: All component IDs MUST contain a hyphen (e.g., my-header).

💡 Opt-in AST Splicing: no-hydration

While Coralite preserves host tags by default, you can choose to completely remove them and only render the template's inner HTML (a process called AST Splicing) by adding the no-hydration attribute to your component instance.

⚠️ Styling the Host Tag

Even though host tags are preserved, Coralite's Scoped CSS system automatically targets elements inside your template. Any styles you write in a component's <style> block are nested under a unique data selector that is applied to the template's children. If you need to style the component "outer shell", it is best to wrap your template content in a container element (like a <div>) and style that.

Static components are perfect for layout structures, simple UI elements, and reusable content blocks. However, when you need complex logic, computed state, or client-side interactivity, you need to upgrade your component to a Dynamic Component.
